Basics

Before You Start

If you haven't already, create a github account, it's free!

While github has made a downloadable GUI program to do all this stuff in, we are going to use the command line. Ill be using the terminal in OS X, but git will work with Windows too. Go to the git website and Download it.


Terminology

    Repository

A Repository or “Repo” is the space in Github where your project will live. The main project files are under the “Master” branch in the repo. When others contribute to the project, the will “branch” off of, making changes under their own branch. When finished, they “merge” there changes into the master.

    Version Control

Version control is the general purpose of Github. Instead of having multiple saves, or overwriting a bunch of times, Github will remember what was changed or added so you never have to worry about losing work if a mistake was made.

    Commit

The commit function is version control in action. When you commit work to your repo, Github will remember what changes you have made, like a checkpoint of your repo. Like in a video game, if you reach a checkpoint, but die 2 minutes later, you can always revert back without having to restart the entire game or pick up from an old save file, loosing all your items and progress. Same premise applies here.


Commands

    Navigation

cd - Change directory

cd .. - Go back one directory


    Setting Up

git help - This will show you the most common git fuctions incase you forget

git config - Use for first time setups, but not needed for this tutorial

git init - Turns an ordinary folder directory into into a Git ready state.


    Working With Your Repo

git status - Will see the status of everything within that git ready local directory

git add - This is the first step of making changes. It does not add the file to the repo, just brings it to Git’s attention and is part of the checkpoint process

git remote -v - Checks the where repos are being pushed, origins and streams

git commit - An important command. It initializes the checkpoint phase of your repo after you make changes. It’s generally to leave a message on every commit so yourself and others know what you did so use its like this: git commit -m “ Type your message here”. The -m indicates that this part of the command will be read as a message

git push - If you are working locally and want your changes to be made visible on Github, use this command to “push” the changes up into the repository

git checkout - This is a navigational command that lets you see inside of other repos or branches that you are not currently inside. You can use it like so: git checkout master or git checkout branch corn to look at another branch

git branch - Use this command when working with others on a project. You can make your own changes and will show you a timeline of commits just incase something goes wrong. Use it like: git branch corn - where corn is the name of the branch you are creating

git pull - When you want the most up-to-date version of your collaborative repo, you “pull” the changes people made down from Github to your local folder

git clone - Makes a copy of selected repository into your local directory

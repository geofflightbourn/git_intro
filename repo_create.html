Setting Up Your Repo

Creating The Repository

Go to github.com and make an account, it's free! Once thats done and your at the homepage, on the left theres a "Your Repositories" box and inside it is a big green button that says "New Repository." Go ahead and click that.

On the next page, give your repository a name; It should be the name of your project or something similar. Choose weather to set it to private or public and don't worry about making readme right now. Go ahead and create the new repo.

Your Local Repository

Now that your repository is setup, it's time now get your local project directory ready to be paired to the online project repository. If you have a project folder already created, good, just make sure it's somewhere easily acessable. If not, create a new folder on your desktop called "git" and inside that one, create a another folder. For the puropses of this demo, just call it "README". Inside your test folder, just create a new .txt file. I'll be using this websites project folder and files.

If you haven't already, read the Basics section of this site before you continue so you can get familiar with some of the commands. Now it's time to use the command line. To start the link between your computer and git, you have to let git know who your are first. In the console, type:

git config --global user.name "username"

Replace "username" with whatever username you just created or already have. Do the same with your email like this:

git config --global user.email "user@mail.com"



Now for a little navigation, we need be inside the project folder we want to link with our git repo. Hopefully your project is in a easy to reach place. For me, it will be like this:

cd desktop
cd git
cd git_intro

The command cd means what it does, changes directory. For me, my project lives indide a folder called "git" on my desktop, for easy navigation.


So here we are inside our folder, but git doesn't know about it yet. To get git talking with our project, we need to initalize communication. We know it's a git command because it starts with git So we go:

git init

So heres what we just did. My directory has alreay been initialized, but yours should say Initalized empty git repository in wherever your directory is.



Nice! Now we can start adding the contents of the folder to our github repository. To see what's in our folder, and the status of our files, we can use the git status command.

It shoudl say your are in your master branch, which is where we want to be. It will also say stuff about Untracked files and be pointing to your text.txt file. This is all fine. Git knows what is, and what isnt, inside the repo and knows that changes have not been made thus far.

We want to make git notice that file, we should go ahead and add that .txt file. Go ahead and do:

git add README.txt

The second step is to commit the file, the checkpoint in the version control system. Type:

git commit -m "your message"

Its good practice to always include a short message with your commits, to tell other people and yourself what changes have been made incase of a conflict or errors.



Connecting the Repos

To this point, we have two repos; local, and github. We need to tell git that we are going to be pushing this local repo up into our github repo, but it needs to know where.

If you go back into your repository on the github website, there will be a coppyable https link at the top. Grab that and go back into terminal to type then paste your link:

git remote add origin https://github.com/geofflightbourn/git_intro.git

Using git remote -v will let you know all the origins (origina locations of files) your github repo knows about. Right now, we only have one, the repo we just connected to moments ago.

So now, all thats left to do is push our files from our local repository, up into the online repository on github. So we just do:

git push origin master

We do this to specify that we are push up into our master branch, our only branch.



Done! If you go check your github repo you should see the file we just added. Congrats!
